# PredProfCanteen

Веб-приложение для автоматизации учета и контроля питания в школьной столовой. Система предоставляет различные интерфейсы для учеников, поваров и администраторов, упрощая процессы заказа, оплаты, выдачи пищи, управления запасами и анализа данных.

## Функциональность

### Для учеников
*   Регистрация и авторизация.
*   Просмотр меню завтраков и обедов.
*   Оплата питания (разовая или по абонементу).
*   Отметка о получении заказа.
*   Указание пищевых аллергий и предпочтений.
*   Оставление отзывов о блюдах.

### Для поваров
*   Авторизация в системе.
*   Учет выданных порций.
*   Контроль остатков продуктов и готовых блюд.
*   Формирование заявок на закупку продуктов.

### Для администраторов
*   Авторизация в системе.
*   Просмотр статистики по оплатам и посещаемости.
*   Согласование заявок на закупки от поваров.
*   Формирование отчетов по питанию и финансовым затратам.

## Технологический стек

*   **Backend:** Python 3.8+, Flask
*   **Frontend:** HTML, CSS (шаблонизатор Jinja2)
*   **База данных:** SQLite3
*   **Библиотеки:** werkzeug.security, datetime, sqlite3, os

## Инструкция по запуску проекта School Canteen
Предварительные требования

Перед началом убедитесь, что на вашем компьютере установлен Docker Desktop. Если Docker не установлен, скачайте и установите его с официального сайта.
Пошаговая инструкция
1. Запуск Docker Desktop

Запустите приложение Docker Desktop и дождитесь его полной инициализации (системный трей должен показывать, что Docker работает).
2. Клонирование репозитория

Откройте терминал (командную строку) и выполните следующую команду для клонирования проекта:
bash

git clone https://github.com/Fluffenn/PredProfCanteen.git

3. Переход в директорию проекта

Перейдите в склонированную директорию:
bash

cd PredProfCanteen

4. Сборка Docker-образа

Выполните сборку Docker-образа с именем school-canteen:
bash

docker build -t school-canteen .

5. Запуск контейнера

Запустите собранный образ, пробросив порт 5000 на хост-машину:
bash

docker run --rm -p 5000:5000 school-canteen

Доступ к приложению

После успешного запуска веб-приложение будет доступно по одному из адресов:

    http://127.0.0.1:5000

    http://172.17.0.2:5000

Рекомендуется сначала проверить доступность по http://127.0.0.1:5000.
Примечания

    Флаг --rm в команде запуска автоматически удаляет контейнер после его остановки

    Порт 5000 на хост-машине будет связан с портом 5000 внутри контейнера

    Для остановки приложения используйте сочетание клавиш Ctrl+C в терминале
